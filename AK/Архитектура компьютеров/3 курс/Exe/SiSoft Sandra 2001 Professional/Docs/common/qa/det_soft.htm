SiSoft Sandra - The Diagnostic Tool, Q & A - Software Detection

This document provides some frequently asked questions about SiSoft Sandra. Please read the Help File as well!

Q: Sandra tells me that the SAN16.EXE (SiSoft Sandra Support) module/process is obsolete! Is this true?
A: No. The version information of this module is 3.1 instead of 4.0 for compatibility with Windows NT. Theoretically it is considered obsolete but practically there is no other way. NT does not support new type 16-bit programs. Update to Sandra version 4.20 or later.

Q: Some 32-bit programs are shown as 16-bit in version information. Is this a bug?
A: When Sandra uses the version checking APIs it does not check itself the type of file it is dealing with. Microsoft forgot to update the version information of some Windows 95 applications (Media Player, Sound Recorder, etc.) to say that they are now 32-bit not 16-bit.

Q: Processor load shows 100% during CPU benchmark. Is this correct?
A: Yes! If this is lower some other programs are also using the CPU and you'll get lower scores.
