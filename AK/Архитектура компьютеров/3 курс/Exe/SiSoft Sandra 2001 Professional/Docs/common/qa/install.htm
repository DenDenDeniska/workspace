SiSoft Sandra - The Diagnostic Tool, Q & A - Installation

This document provides some frequently asked questions about SiSoft Sandra. Please read the Help File as well!

Q: Sandra is linked to missing export in MFC42. Program crashes
Q: Sandra does not start at all
A: This means the MFC42 (Microsoft Class) libraries are not up to date. A Properties on the MFC42.DLL file in \Windows\System (use Start - Find if you cannot find it) should return a version of 6.00.8168.0 or later (preferably 6.00.8267.0). Try one of the following:

  1. For more information read Microsoft KB Article Q197298.
  2. Windows Patch
    1. Main Download Page @ Microsoft
    2. Windows 98. Use Start - Windows Update and apply the Microsoft Libraries Update
    3. Windows 95. Download Windows Library Update patch from Updates for Windows 95
  3. Download and install Sandra Standard 99 (in a different folder) and then uninstall it. The installer is more advanced and should be able to install the updated libraries.
  4. In anger, you can try installing Microsoft Internet Explorer 5 or later.
  5. We can e-mail them to you. This is riskier since you just copy them rather than going through a proper update. Also the file is rather large.

NB 1: We have received reports that the patches are not updating the system libraries correctly. In this case delete it from your drive and try again. Try a different option if one option fails.

NB 2: If you're successfully updated the libraries (check whether the version matches), you may have to update other system libraries as well - see the issues below.

Q: SANDRA caused an invalid page fault in module MFC42.DLL. Program crashes
Q: Sandra does not start at all
A: This means the ComCtl32 (common controls) libraries are not up to date. A Properties on the ComCtl32.DLL file in \Windows\System (use Find if you cannot find it) should return a version of 5.80 or later (product version 5.00.2014.216). Try one of the following:

  1. Windows Patch
    1. Main Download Page @ Microsoft
    2. Windows 98. Use Start - Windows Update and apply the Microsoft Libraries Update
    3. Windows 95. Download Common Controls Update from Updates for Windows 95 or Visual C++ Downloads
  2. In anger, you can try installing Microsoft Internet Explorer 5 or later.
  3. We can e-mail them to you. This is riskier since you just copy them rather than going through a proper update. Also the file is rather large.

NB 1: We have received reports that the patches are not updating the system libraries correctly. In this case delete it from your drive and try again. Try a different option if one option fails.

NB 2: We've also learned that some patches do not contain the latest version of the system libraries as we have thought - depending on download server. Try a different option if one option fails.

Q: Some programs from the Tools menu are missing!
A: These programs are not included in the package. They either come with Windows or you need to download them. To find out - click F1 after you've selected the menu option or start by clicking the Help button on the toolbar and then select the menu option.

Q: Sandra Professional Patch says Version & Type does not match any known version
A: Uninstall All copies of Sandra. Install a fresh copy of the Professional version. Try to patch it again.

Q: When I try to patch nothing is downloaded and the program exits
Q: When I try to patch a small (1K) file is downloaded and I get "program is invalid".
A: Look into the temporary folder for a file called Update.exe. If the file is small, open it with Notepad (hold Shift key and right-click, choose Open With and select Notepad). The file may contain a web error which was not caught by Sandra. The error message should identify the actual error.

Q: Sandra has a problem with the Internet! (WININET)
A: Uninstall Sandra. If you have MS-IE 4.0 or later installed, you may need to reinstall IE. If not, delete the WININET.DLL file in the \Windows\System folder. Then reinstall Sandra. Sandra will install a fresh copy of this library.
